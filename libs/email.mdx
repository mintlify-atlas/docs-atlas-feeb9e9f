title: "Email"
description: "Send transactional emails via SMTP using the Email class, which wraps PHPMailer."
---

The `Email` class (`libs\Middle\Email`) provides a single static method for sending HTML emails over SMTP. It wraps [PHPMailer](https://github.com/PHPMailer/PHPMailer) and reads its configuration from `config/email.php`.

## Configuration

Email settings are defined in `config/email.php` and populated from environment variables.

```php
// config/email.php
return [
    'host'     => env('EMAIL_HOST'),
    'port'     => env('EMAIL_PORT'),
    'username' => env('EMAIL_USERNAME'),
    'password' => env('EMAIL_PASSWORD'),
    'from'     => env('EMAIL_FROM'),
];
```

Set the corresponding variables in your `config.env` file:

```bash
EMAIL_HOST=smtp.example.com
EMAIL_PORT=587
EMAIL_USERNAME=no-reply@example.com
EMAIL_PASSWORD=your-smtp-password
EMAIL_FROM=no-reply@example.com
```

<Note>
The `Email` class uses `STARTTLS` encryption (`PHPMailer::ENCRYPTION_STARTTLS`) on the configured port. Port `587` is the standard STARTTLS port. Use port `465` with `SMTPS` if your provider requires SSL instead — you will need to adjust the `SMTPSecure` setting in `libs/Middle/Email.php` in that case.
</Note>

## Sending an email

```
Email::send(string $email, string $title, string $subject, string $body): bool
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `$email` | `string` | Recipient email address |
| `$title` | `string` | Display name of the sender (shown in the "From" field) |
| `$subject` | `string` | Email subject line |
| `$body` | `string` | HTML body of the email |

Returns `true` on success and `false` on failure. Exceptions thrown by PHPMailer are caught internally.

### Basic example

```php
use libs\Middle\Email;

$sent = Email::send(
    'user@example.com',
    'My App',
    'Welcome to My App',
    '<h1>Welcome!</h1><p>Your account has been created.</p>'
);

if (!$sent) {
    // Log the failure or return an error response
    abort(500, 'Failed to send email');
}
```

## HTML emails

The `$body` parameter is always treated as HTML (`$mail->isHTML(true)`). You can pass a full HTML document or a partial HTML fragment.

```php
$body = <<<HTML
<!DOCTYPE html>
<html>
<body style="font-family: sans-serif; color: #333;">
    <h2>Password reset</h2>
    <p>Click the link below to reset your password. This link expires in 30 minutes.</p>
    <p><a href="{$resetUrl}" style="color: #4F46E5;">Reset your password</a></p>
    <p>If you did not request a password reset, you can ignore this email.</p>
</body>
</html>
HTML;

$sent = Email::send(
    $user->email,
    'My App',
    'Reset your password',
    $body
);
```

<Tip>
For reusable email templates, render a view to a string with output buffering and pass the result as `$body`:

```php
ob_start();
include base_path('resources/views/emails/welcome.php');
$body = ob_get_clean();

Email::send($user->email, 'My App', 'Welcome', $body);
```
</Tip>

## Using in a controller

```php
use libs\Middle\Email;

class AuthController
{
    public function register($request, $response)
    {
        // ... create the user ...

        Email::send(
            $request->input('email'),
            'My App',
            'Verify your email address',
            '<p>Thank you for registering. <a href="' . $verifyUrl . '">Verify your email</a></p>'
        );

        return $response->redirect('/login');
    }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Email is not being sent">
    - Verify `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USERNAME`, and `EMAIL_PASSWORD` in `config.env`.
    - Confirm your SMTP server accepts connections on the configured port.
    - Check that `EMAIL_FROM` matches an address authorized on your SMTP server.
    - Some providers (e.g. Gmail) require an app-specific password when 2FA is enabled.
  </Accordion>
  <Accordion title="SMTP debug output appears in the response">
    `Email::send()` wraps the PHPMailer call in `ob_start()` / `ob_end_clean()` to suppress SMTP debug output (`SMTP::DEBUG_SERVER`). If debug output leaks, ensure output buffering is active before `Email::send()` is called.
  </Accordion>
  <Accordion title="Special characters are garbled">
    The body is passed through `utf8_encode()` before being handed to PHPMailer. Make sure your source string is already UTF-8 encoded to avoid double-encoding issues.
  </Accordion>
</AccordionGroup>
