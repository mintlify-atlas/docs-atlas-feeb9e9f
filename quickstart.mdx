This guide takes you from zero to a running MVC PHP app. You'll clone the project, configure your environment, run migrations, and open your first page in the browser.

## Prerequisites

Before starting, make sure you have:
- PHP ≤ 8.1
- Composer
- MySQL or MariaDB
- Git

<Steps>
  <Step title="Clone the repository">
    Clone MVC PHP and enter the project directory:

    ```bash
    git clone https://github.com/dogiloki/mvc-php.git
    cd mvc-php
    ```
  </Step>

  <Step title="Install PHP dependencies">
    Install all Composer packages:

    ```bash
    composer install
    ```

    This installs Guzzle, PHPMailer, Intervention Image, Faker, scssphp, and more.
  </Step>

  <Step title="Configure your environment">
    Copy the example environment file and fill in your values:

    ```bash
    cp config.env.example config.env
    ```

    Open `config.env` and update the database credentials at minimum:

    ```ini config.env
    APP_NAME=localhost
    APP_KEY=your-secret-key
    APP_HOST=localhost
    APP_DEBUG=true

    DB_HOST=localhost
    DB_PORT=3306
    DB_USER=root
    DB_PASSWORD=your_password
    DB_DATABASE=mvc_php
    ```

    <Note>
      Alternatively, run `php manager create` to auto-generate `config.env` from
      `config.env.example` along with any other required initial files.
    </Note>
  </Step>

  <Step title="Run migrations">
    Create your database tables by running the migrations:

    ```bash
    php manager migrate
    ```

    This executes all migration files found in `database/migrations/` in order.
  </Step>

  <Step title="Start the development server">
    Launch the built-in PHP server pointed at the `public/` directory:

    ```bash
    php manager server
    ```

    The server starts on `http://127.0.0.1:8000` by default. Open that URL in your browser — you should see the home page.

    <Tip>
      To use a different port, pass it as an argument: `php manager server 9000`
    </Tip>
  </Step>
</Steps>

## Your first route

Open `routes/web.php`. You'll see the default route that renders the home view:

```php routes/web.php
<?php

use libs\Router\Route;

Route::get('/', function() {
    return view('home');
})->name('home');
```

Add a new route below it:

```php routes/web.php
Route::get('/hello', function() {
    return view('hello', ['name' => 'World']);
})->name('hello');
```

Then create the view file at `resources/views/hello.php`:

```php resources/views/hello.php
<h1>Hello, <?= $name ?>!</h1>
```

Visit `http://127.0.0.1:8000/hello` to see it in action.

## Your first controller

Use the CLI to scaffold a controller:

```bash
php manager new controller HelloController
```

This creates `app/Controllers/HelloController.php`. Edit it to add an action:

```php app/Controllers/HelloController.php
<?php

namespace App\Controllers;

use libs\HTTP\Request;

class HelloController {

    public function index(Request $request) {
        return view('hello', ['name' => $request->get('name') ?? 'World']);
    }

}
```

Then point a route at it:

```php routes/web.php
Route::get('/hello', 'HelloController@index')->name('hello');
```
